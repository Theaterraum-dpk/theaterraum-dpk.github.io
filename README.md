# 테아터라움 철학하는 몸 — 극단 홈페이지

실험극단 **테아터라움 철학하는 몸**의 정적 홈페이지입니다.
순수 HTML / CSS / JS로만 제작되어 있어 별도의 서버나 빌드 과정 없이
GitHub Pages 등 정적 호스팅에 바로 올릴 수 있습니다.

## 폴더 구조

```
theater-site/
├─ index.html      홈 (히어로, 극단 태도, 최근 작품 미리보기)
├─ about.html       극단소개 (연출, 배우진, 연혁)
├─ works.html        작품 (레퍼토리)
├─ board.html        평론 게시판
├─ css/style.css     전체 디자인 시스템
├─ js/main.js         내비게이션 · 스크롤 애니메이션
└─ js/board.js         게시판 로직 (localStorage 기반)
```

## GitHub Pages로 배포하기

1. 이 폴더 전체를 새 GitHub 저장소에 푸시합니다.
2. 저장소 **Settings → Pages**로 이동합니다.
3. **Branch**를 `main` (또는 사용 중인 브랜치), 폴더는 `/ (root)`로 설정하고 저장합니다.
4. 몇 분 뒤 `https://[사용자명].github.io/[저장소명]/` 주소로 접속하면 사이트가 열립니다.

## 내용 수정하기

- **극단 이름 / 연출 / 배우** — `about.html`의 "연출"과 "소속 배우" 섹션을 실제 정보로 바꿔주세요.
  현재 허지윤 외 배우 3명(박세온, 윤도경, 강민아)은 예시로 채워 넣은 이름입니다.
- **작품 목록** — `works.html`과 `index.html`의 카드 내용을 실제 공연 정보로 교체하세요.
- **연락처 / 주소 / SNS** — 각 페이지 하단 `<footer>`의 이메일, 주소, 인스타그램 링크를 실제 정보로 바꿔주세요.
- **색상 / 폰트** — `css/style.css` 상단 `:root` 안의 CSS 변수(`--bg`, `--ink`, `--red` 등)만 바꾸면
  전체 사이트의 톤이 함께 바뀝니다.

## 평론 게시판에 대해 (중요)

`board.html`의 게시판은 **브라우저 localStorage**에 글을 저장하는 방식으로 구현되어 있습니다.
그래서 서버 없이도 GitHub Pages에서 바로 "글쓰기 → 목록에 뜨는" 동작을 확인할 수 있지만,
아래 두 가지 한계가 있습니다.

- 방문자 A가 쓴 글은 **방문자 A의 브라우저에만** 저장됩니다. 방문자 B는 그 글을 볼 수 없습니다.
- 브라우저 저장소를 지우거나 다른 기기로 접속하면 글이 사라집니다.

즉 지금 상태로는 "나만 보는 메모장" 정도의 데모이고, **모든 방문자가 함께 보는 진짜 게시판**은 아닙니다.
실제 서비스로 쓰려면 아래 중 한 가지 방법으로 백엔드를 연결하는 것을 권장합니다.

### 게시판을 실제로 공유하려면

1. **giscus (추천, GitHub 저장소만으로 가능)**
   깃허브 저장소의 Discussions 기능을 댓글창처럼 쓰는 오픈소스 위젯입니다.
   별도 서버 없이 GitHub Pages와 함께 그대로 동작하며, 방문자들의 글이 실제로 공유됩니다.
   https://giscus.app 에서 저장소를 연결하고 안내에 따라 발급되는 `<script>` 태그를
   `board.html`의 게시판 영역에 붙여 넣으면 됩니다. (GitHub 로그인 필요)

2. **Firebase / Supabase 같은 무료 백엔드 서비스**
   실제 데이터베이스에 글을 저장하고, 로그인 없이도 익명으로 글을 쓸 수 있게
   구성할 수 있습니다. `js/board.js`의 `loadPosts`, `savePosts` 함수를
   해당 서비스의 API 호출로 바꿔주면 지금의 화면 디자인은 그대로 유지할 수 있습니다.

3. **정적 사이트 + 폼 서비스 (예: Formspree, Google Forms)**
   글쓰기 폼 제출만 외부 서비스로 보내고, 관리자가 검수 후 게시물을 페이지에
   직접 추가하는 간단한 운영 방식입니다.

당장은 데모 상태로 두고, 정식 오픈 전에 위 방법 중 하나를 연결하는 것을 추천드립니다.

## 브라우저 지원

최신 Chrome, Safari, Edge, Firefox 기준으로 확인했습니다. `prefers-reduced-motion`을
켜 둔 경우 애니메이션이 자동으로 최소화됩니다.
